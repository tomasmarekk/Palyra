Palyra operator CLI for secure local and remote runtime management.

Usage: palyra [OPTIONS] <COMMAND>

Commands:
  version         Show CLI build metadata
  setup           Bootstrap a Palyra installation [aliases: init]
  doctor          Run diagnostics, repair previews, and rollback workflows
  health          Run a narrow liveness/readiness probe across HTTP and gRPC gateway surfaces
  logs            Tail local gateway journal diagnostics
  status          Show transport and admin status across HTTP/gRPC surfaces
  acp             Run the ACP stdio bridge and legacy compatibility shim
  agent
  agents
  routines        Manage unified automation routines across schedules and event triggers
  cron            Manage schedule-only cron workflows through the routines compatibility layer
  memory
  message
  approvals
  sessions
  tui             Launch the terminal operator client
  auth
  channels
  webhooks        Manage webhook-backed integrations
  docs            Discover bundled or committed operator docs and CLI help snapshots locally
  plugins
  hooks
  devices
  node
  nodes
  browser         Operate the browser service and browser-backed automation sessions
  system          Inspect runtime heartbeat, subsystem presence, and recent system events
  sandbox         Inspect effective process-runner and WASM sandbox policy surfaces
  completion      Generate shell completion scripts
  onboarding      [aliases: onboard]
  configure       Safely reconfigure an existing installation
  gateway         Gateway and runtime diagnostics surface [aliases: daemon]
  dashboard       Resolve or open the operator dashboard URL
  backup          Create or verify portable operator backups
  reset           Reset selected local runtime scopes
  uninstall       Remove an installed Palyra package and optional state
  update          Check or apply a packaged Palyra update
  support-bundle
  policy
  protocol
  config
  models
  patch
  skills          [aliases: skill]
  secrets
  security
  tunnel
  pairing

Options:
  -h, --help
          Print help (see a summary with '-h')

Global Options:
      --profile <PROFILE>

      --config <CONFIG_PATH>

      --state-root <STATE_ROOT>

  -v, --verbose...
          Increase logging verbosity (-v => debug, -vv => trace)

      --log-level <LOG_LEVEL>
          [default: info]
          [possible values: error, warn, info, debug, trace]

      --output-format <OUTPUT_FORMAT>
          [default: text]
          [possible values: text, json, ndjson]

      --plain

      --no-color

Examples:
  palyra setup --mode local
  palyra acp --session-key ops:triage
  palyra docs search acp
  palyra gateway status
  palyra dashboard --open
  palyra backup create --output ./artifacts/palyra-backup.zip
  palyra system heartbeat
  palyra sandbox explain --runtime process-runner
  palyra update --check
  palyra --profile staging agents list --json
  palyra --config ./palyra.toml --output-format json status --admin

Canonical command map:
  setup      Preferred bootstrap/init workflow (`init` remains as a compatibility alias)
  configure  Guided reconfiguration workflow for an existing installation
  acp        Preferred ACP stdio bridge entry point (`agent acp` remains compatible)
  docs       Local committed docs/help discovery surface
  gateway    Preferred runtime/admin family (`daemon` remains as a compatibility alias)
  dashboard  Thin operator shortcut for dashboard URL discovery/open workflows
  routines   Unified automation surface for schedules, hooks, webhooks, and system events
  backup     Portable lifecycle backup/create verification surface
  system     Runtime heartbeat, presence, and recent system-event observability
  sandbox    Effective isolation/runtime policy explain surface for process and WASM tooling
  plugins    Trusted WASM plugin binding and lifecycle surface
  hooks      Event-driven automation bindings over trusted plugins
  reset      Destructive local recovery surface with explicit scope selection
  uninstall  Installer-aware package removal surface
  update     Package update/check orchestration surface
  onboarding Operator onboarding workflows (`onboard` stays as the shorthand alias)
  webhooks   Webhook-backed integration management surface
