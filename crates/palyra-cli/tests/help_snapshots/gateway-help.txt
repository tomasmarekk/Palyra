Gateway and runtime diagnostics surface

Usage: palyra gateway [OPTIONS] <COMMAND>

Commands:
  run
  health
  probe
  discover
  call
  usage-cost
  install
  start
  stop
  restart
  uninstall
  logs
  status
  admin-status
  journal-recent
  journal-vacuum
  journal-checkpoint
  run-status
  run-tape
  run-cancel
  dashboard-url

Options:
  -h, --help
          Print help (see a summary with '-h')

Global Options:
      --profile <PROFILE>

      --config <CONFIG_PATH>

      --state-root <STATE_ROOT>

  -v, --verbose...
          Increase logging verbosity (-v => debug, -vv => trace)

      --log-level <LOG_LEVEL>
          [default: info]
          [possible values: error, warn, info, debug, trace]

      --output-format <OUTPUT_FORMAT>
          [default: text]
          [possible values: text, json, ndjson]

      --plain

      --no-color

Examples:
  palyra gateway run
  palyra gateway health
  palyra gateway probe
  palyra gateway discover --verify-remote
  palyra gateway call health
  palyra gateway usage-cost --days 7
  palyra gateway status
  palyra gateway admin-status --token <token>
  palyra gateway dashboard-url --verify-remote --open

Discoverability:
  `palyra dashboard` is the thin shortcut for dashboard URL workflows.
