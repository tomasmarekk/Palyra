Safely reconfigure an existing installation

Usage: palyra configure [OPTIONS]

Options:
      --path <PATH>

      --section <SECTIONS>
          [possible values: workspace, auth-model, gateway, runtime-controls, daemon-service, channels, skills, health-security]

      --non-interactive

      --accept-risk

      --json

      --workspace-root <WORKSPACE_ROOT>

      --auth-method <AUTH_METHOD>
          [possible values: api-key, anthropic-api-key, skip, existing-config]

      --api-key-env <API_KEY_ENV>

      --api-key-stdin

      --api-key-prompt

      --bind-profile <BIND_PROFILE>
          [possible values: loopback-only, public-tls]

      --daemon-port <DAEMON_PORT>

      --grpc-port <GRPC_PORT>

      --quic-port <QUIC_PORT>

      --tls-scaffold <TLS_SCAFFOLD>
          [possible values: none, bring-your-own, self-signed]

      --tls-cert-path <TLS_CERT_PATH>

      --tls-key-path <TLS_KEY_PATH>

      --remote-base-url <REMOTE_BASE_URL>

      --admin-token-env <ADMIN_TOKEN_ENV>

      --admin-token-stdin

      --admin-token-prompt

      --remote-verification <REMOTE_VERIFICATION>
          [possible values: none, server-cert, gateway-ca]

      --pinned-server-cert-sha256 <PINNED_SERVER_CERT_SHA256>

      --pinned-gateway-ca-sha256 <PINNED_GATEWAY_CA_SHA256>

      --ssh-target <SSH_TARGET>

      --skip-health

      --skip-channels

      --skip-skills

  -h, --help
          Print help (see a summary with '-h')

Global Options:
      --profile <PROFILE>

      --expect-profile <EXPECT_PROFILE>

      --config <CONFIG_PATH>

      --state-root <STATE_ROOT>

  -v, --verbose...
          Increase logging verbosity (-v => debug, -vv => trace)

      --log-level <LOG_LEVEL>
          [default: info]
          [possible values: error, warn, info, debug, trace]

      --output-format <OUTPUT_FORMAT>
          [default: text]
          [possible values: text, json, ndjson]

      --plain

      --no-color

      --allow-profile-mismatch

      --allow-strict-profile-actions

Examples:
  palyra configure
  palyra configure --section workspace --section auth-model
  palyra configure --non-interactive --section gateway --bind-profile public-tls --accept-risk

Discoverability:
  `configure` reuses the onboarding wizard engine to safely edit an existing installation.
