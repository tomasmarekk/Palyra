Inspect effective process-runner and WASM sandbox policy surfaces

Usage: palyra sandbox [OPTIONS] <COMMAND>

Commands:
  list
  explain

Options:
  -h, --help
          Print help (see a summary with '-h')

Global Options:
      --profile <PROFILE>

      --config <CONFIG_PATH>

      --state-root <STATE_ROOT>

  -v, --verbose...
          Increase logging verbosity (-v => debug, -vv => trace)

      --log-level <LOG_LEVEL>
          [default: info]
          [possible values: error, warn, info, debug, trace]

      --output-format <OUTPUT_FORMAT>
          [default: text]
          [possible values: text, json, ndjson]

      --plain

      --no-color

Examples:
  palyra sandbox list
  palyra sandbox explain --runtime process-runner
  palyra sandbox explain --runtime wasm-runtime --json

Discoverability:
  `sandbox` reads the effective runtime policy snapshot from admin diagnostics. Use `palyra policy explain` for per-action Cedar decisions.
